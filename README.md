# MSPM0G3507 八路灰度循迹测试

这是一个最小的白底黑线 P 控制循迹程序。八路传感器通过 CD4051 的 `AD2/AD1/AD0` 选通，`OUT` 由 `PA24/ADC0.3` 读取。

## 引脚

| 功能 | MSPM0G3507 |
| --- | --- |
| AIN1 / AIN2 | PA13 / PA12 |
| BIN1 / BIN2 | PB0 / PB8 |
| PWMA / PWMB | PB4 / PB1 |
| E1A / E1B | PA15 / PA16 |
| E2A / E2B | PA17 / PB9 |
| 传感器 OUT | PA24 (ADC0.3) |
| 传感器 AD2 / AD1 / AD0 | PB24 / PB25 / PB20 |

传感器 `EN` 可悬空，内部 10 kOhm 下拉会保持使能。MCU、传感器和电机驱动必须共地。传感器使用稳定的 5 V 电源，并与电机电源做好隔离/滤波。上电前用万用表确认传感器 `OUT` 不会超过 MCU 的 `VDDA`（通常 3.3 V）；若超过，必须先加分压或限幅，不能直接接 `PA24`。

## 首次测试

1. 先把车轮架空，不要直接放到地面运行。
2. 复位后电机强制停止 3 秒；在此期间把全部 8 个探头移到白底。随后保持白底约 1 秒，程序自动记录每一路白底值。
3. 白底标定后，将小车移到赛道起点，让第 4/5 个中心探头压住黑线、最外侧探头保持在白底。中心线连续稳定约 0.2 秒后，再等待 0.5 秒自动起步。
4. 运行时程序持续学习每路黑线最小值。默认 `BASE_SPEED=300`（30% PWM），P 增益为 60；丢线或转弯超时会停车，重新检测到线后可以恢复，不再永久锁死。
5. 轮子架空时确认两轮的正转方向。某一轮反转就在源码顶部把对应的 `LEFT_MOTOR_REVERSED` 或 `RIGHT_MOTOR_REVERSED` 改为 1。

程序假设第 1 路传感器位于车体左侧，第 8 路位于右侧。如果实物顺序相反，将 `gSensorPosition[]` 的符号/顺序反过来。

编码器引脚已配置为数字输入，但这个最小测试暂时不做编码器速度闭环。

## 直角弯状态机

正常直线使用加权 P 控制。当左侧或右侧连续三个探头检测到横向黑线时，程序进入对应的直角转弯状态。转弯过程中外轮低速前进、内轮轻微反转；只有中心探头经历一次丢线并重新检测到黑线后，程序才恢复直线控制。若约 1 秒仍未重新找到线，电机会停止。

可调参数位于源码顶部：`TURN_OUTER_SPEED` 是转弯外轮速度，`TURN_INNER_SPEED` 是内轮速度；直角转不过去可增大外轮速度或加大内轮反转，转过头则反向调整。程序同时支持左直角和右直角。

## nanoDAP 无线烧录

工程包含共享 External Tool 配置 `nanoDAP_Build_and_Flash.launch`。在 CCS 中右键该文件并选择 `Run As > nanoDAP Build and Flash`；也可以从工具栏的 External Tools 历史中再次运行。它会先构建 Debug 配置，再通过 CMSIS-DAP/OpenOCD 以兼容模式和 100 kHz SWD 烧录、校验并复位运行。

CCS 20/Theia 用户使用工作区 `E:\download\yun\.vscode\tasks.json`：按 `Ctrl+Shift+P`，运行 `Tasks: Run Task`，再选择 `nanoDAP: Build and Flash MSPM0G3507`。

烧录前确认无线发射端和接收端蓝灯均常亮，并连接 `GND`、`3.3V/VTref`、`SWDIO`、`SWCLK`，建议同时连接 `nRST`。首次运行电机程序时必须架空车轮。

# 改造记录

## 目标
MSPM0G3507 八路灰度循迹小车，将传感器 ADC 读取改为 **DMA burst 搬运**。

## 仓库
GitHub: https://github.com/giaoLRL/simple-standard-car  
（仓库 slug 为英文 `simple-standard-car`，描述为中文“简易标准”）

## 当前状态
- ADC 配置为 repeat-single-channel 模式。
- 每路传感器启动 1 次 DMA，连续搬运 8 个样本到 `g_adc_dma_buf[8]`。
- DMA 完成中断仅触发 1 次，CPU 在中断里停止 ADC 并置标志。
- MUX 切换仍由 CPU 控制（硬件只有 1 个 OUT）。

## 必须人工核对

1. **ADC repeat 模式**：在 SysConfig GUI 里确认 `LINE_ADC` 为 **Repeat Single Channel**，否则只采 1 次。
2. **DMA 触发源**：确认触发源为 ADC0 / ADC12_0 的 MEM0 结果加载事件。
3. **DMA 目标递增**：确认 `dstAddrIncrement` 为 Increment，`transferSize = 8`。
4. **PB21 按键**：在 SysConfig GUI 里确认 `KEY_USER` 为输入、上拉、接 PORTB Pin 21。
5. **PA0 蜂鸣器**：在 SysConfig GUI 里确认 `BUZZER` 为输出、推挽、初始低电平、接 PORTA Pin 0。
6. **重新生成配置**：保存 `.syscfg` 后让 CCS 重新生成 `ti_msp_dl_config.h`。
7. **API 兼容性**：以下函数/宏若编译报错，按 SDK 头文件调整：
   - `DL_ADC12_getMemResultAddress()`
   - `DL_DMA_getPendingInterrupt()` 参数个数
   - `DL_DMA_IIDX_DMA_CHANNEL0_DONE`
   - `DL_ADC12_stopConversion()`
   - `LINE_DMA_INST` / `LINE_DMA_INST_INT_IRQN`

## 已知限制

- MUX 切换仍是 CPU 阻塞操作。
- DMA burst 完成后才唤醒主循环，本质仍是阻塞式，但中断次数从 64 次/周期降到 8 次/周期。
- 已移除运行中动态学习；黑/白基准由 PB21 按键触发上电双标定确定。
- 丢线时停车，蜂鸣器以 1 Hz 间歇鸣叫报警。
- 支持左/右直角弯状态机，检测到横向黑线后差速转向，中心探头重新找线后恢复。
- API 基于 MSPM0 DriverLib 常见命名，不同 SDK 版本可能需要微调。

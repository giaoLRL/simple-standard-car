# 速度环（Velocity Loop）方案设计

> 版本：v1.0 | 状态：方案阶段 | 目标：在现有差速转向架构上增加速度闭环，对接小程序且不影响现有功能

---

## 一、目标

在现有**开环 PWM 驱动**的差速转向基础上，增加编码器测速 + 双轮速度 PID 闭环，形成**级联控制**架构：

```
方向 PID（现有，10ms）→ 目标左右轮速 → 速度 PID（新增，5~10ms）→ 电机 PWM
```

---

## 二、控制架构对比

### 现有架构（开环）

```
line_sensor → error → 位置式 PID → correction
                                         ↓
                          left_PWM  = baseSpeed + correction
                          right_PWM = baseSpeed - correction
                                         ↓
                                   Motor PWM
```

**问题**：电池电压下降、地面摩擦力变化时，相同 PWM 产生的实际轮速不一致，导致循迹性能漂移。

### 新增架构（速度闭环）

```
line_sensor → error → 位置式 PID → correction (转向修正量)
                                         ↓
                          left_target  = baseSpeed + correction
                          right_target = baseSpeed - correction
                                         ↓
                          ┌─ 左轮速度 PID ← encoder_left_ticks
                          │       ↓
                          │  left_PWM  → MotorL
                          │
                          └─ 右轮速度 PID ← encoder_right_ticks
                                  ↓
                             right_PWM → MotorR
```

转向 PID 输出**目标速度**（而非直接输出 PWM），速度 PID 负责**精确跟踪目标速度**。

## 三、硬件要求

### 3.1 编码器选型（MG310 电机标配）

**MG310 霍尔编码器**（已随电机配备）：

| 参数 | 值 | 说明 |
|------|-----|------|
| 类型 | 霍尔磁感应 | 非光电式 |
| 线数 | **13 CPR**（电机轴） | 输出轴 = 13 × 减速比 = 260 脉冲/转 |
| 减速比 | **1:20** | 额定 400RPM → 输出轴 20RPM |
| 供电 | 3.3V~5V | 兼容 MSPM0 3.3V 电平 |
| 接口 | PH2.0 6P | 自带上下拉整形，**MCU 可直接 GPIO 读取** |

> 轮径 48mm，周长 ≈ 150.8mm。每脉冲对应位移 ≈ 150.8/260 ≈ **0.58mm**（空间分辨率很高）。

**低 CPR 带来的挑战**：13 线在额定 400RPM 时仅产生 ~87 脉冲/秒，10ms 采不到 1 个脉冲。必须改用 **T 法（周期测量）**，详见 3.3。

### 3.2 GPIO 分配（已在 syscfg 中配置）

`GPIO_ENCODER` 组（全部 INPUT）：

| 信号 | SysCfg 名称 | 引脚 | 说明 |
|------|------------|------|------|
| ENC_LEFT_A | E1A | PA15 | 左轮编码器 A 相 |
| ENC_LEFT_B | E1B | PA16 | 左轮编码器 B 相 |
| ENC_RIGHT_A | E2A | PA17 | 右轮编码器 A 相 |
| ENC_RIGHT_B | E2B | PB9 | 右轮编码器 B 相 |

> 引脚已配好，无需修改 syscfg。直接用 `DL_GPIO_readPins()` 读取。

### 3.3 测速方案

**必须用 T 法（脉冲周期测量）**，因为 13 CPR 在 10ms 内脉冲数太少：

| 电机转速 | 脉冲频率 | 脉冲间隔 | 10ms M 法脉冲数 |
|----------|----------|----------|-----------------|
| 400 RPM（额定） | 87 Hz | 11.5 ms | 0~1 |
| 200 RPM | 43 Hz | 23 ms | 0~1 |
| 100 RPM | 22 Hz | 46 ms | 0 |

M 法在 10ms 窗口完全不可用。T 法用 Timer 捕获相邻脉冲的时间间隔，低速反而精度高。

**实现方式**：
- PA15(E1A)、PA17(E2A) 配置为 **GPIO 输入中断**（双边沿触发）
- ISR 中记录 `timebase_micros()` 差值 → 即脉冲周期
- 4x 正交解码：A/B 相双边沿都计数，13 CPR → **52 边沿/电机转**

```
motor_rpm = 60,000,000 / (pulse_period_us × 13 × 4)
         = 60,000,000 / (pulse_period_us × 52)
```

或者简化为用 A 相单边沿（不 4x），避免 B 相中断开销：

```
motor_rpm = 60,000,000 / (pulse_period_us × 13)
```

> **速度环周期**：T 法下速度更新频率 = 脉冲到达频率。200RPM 时约 43Hz（~23ms 更新一次）。速度 PID 每个主循环周期（10ms）用最近一次测得的 rpm 值计算，未更新时保持上一次值。

### 3.4 中断优先级分配

MSPM0G3507 (Cortex-M0+) 使用 **GROUP0 + GROUP1** 组合中断架构。GPIOA 和 GPIOB 共享同一个 `GROUP1_IRQHandler`，ISR 内读 IIDX 区分具体端口。

编码器用 PA15 和 PA17（都在 GPIOA），两者共享中断入口，无需分别占用向量。

**当前中断一览**：

| 中断向量 | Handler | 当前用途 | 默认优先级 |
|----------|---------|----------|-----------|
| `SysTick` | `SysTick_Handler` | 1ms 时基 | 0（系统异常，天然最高） |
| `DMA_INT_IRQn` | `DMA_IRQHandler` | ADC DMA burst 完成 | 0（默认） |
| `UART0_INT_IRQn` | `UART_0_INST_IRQHandler` | UART TX/RX | 0（默认） |
| `GROUP1_IRQn` | 未使用 | **编码器 GPIO**（新增） | — |

**优先级分配原则**：T 法对脉冲时间戳精度敏感，ISR 入第一行就应读 `timebase_micros()`。编码器 ISR 极短（<2μs），应设最高 NVIC 优先级以减少捕获抖动。

| 中断 | 建议优先级 | 理由 |
|------|-----------|------|
| SysTick | 0（系统异常） | 时基，天然高于所有 NVIC 中断 |
| **GROUP1（编码器）** | **0（NVIC）** | 捕获时间戳，ISR 极短，应最先响应 |
| DMA_INT | 1 | ADC 数据完整性，10ms 窗口内完成即可 |
| UART0 | 2 | 9600bps ≈ 1ms/字节，容忍延迟最长 |

> Cortex-M0+ NVIC 共 4 级（0~3），0 = 最高。SysTick 是系统异常（SCB），优先级天然高于所有 NVIC 外设中断，编码器 ISR 内读 `SysTick->VAL` 不会与 `SysTick_Handler` 产生竞争。

**GROUP1_IRQHandler 实现**：

```cpp
extern "C" void GROUP1_IRQHandler(void)
{
    uint32_t iidx = DL_Interrupt_getPendingGroup(DL_INTERRUPT_GROUP_1);
    uint32_t now_us = timebase_micros();  // ★ 第一行就抓时间戳

    if (iidx == DL_INTERRUPT_GROUP1_IIDX_GPIOA) {
        uint32_t pending = DL_GPIO_getPendingPinStatus(GPIOA);

        if (pending & DL_GPIO_PIN_15) {
            DL_GPIO_clearInterruptStatus(GPIOA, DL_GPIO_PIN_15);
            g_enc_left.on_pulse(now_us);
        }
        if (pending & DL_GPIO_PIN_17) {
            DL_GPIO_clearInterruptStatus(GPIOA, DL_GPIO_PIN_17);
            g_enc_right.on_pulse(now_us);
        }
    }
}
```

> 只用了 A 相（PA15/PA17），不涉及 PB9(E2B)，无需 GPIOB 分支。将来若加 B 相做方向判断，追加 `else if (iidx == DL_INTERRUPT_GROUP1_IIDX_GPIOB)` 即可。

### 3.5 timebase 扩展：微秒时间戳

当前 `timebase.cpp` 仅有 `timebase_millis()`（1ms 精度），T 法需要微秒级脉冲周期测量。利用 SysTick 递减计数器 `VAL` 反推即可，无需额外硬件定时器：

```cpp
// timebase.hpp / timebase.cpp 新增
uint32_t timebase_micros(void);
```

```cpp
uint32_t timebase_micros(void)
{
    uint32_t ms, val;
    do {
        ms   = g_millis;         // 快照毫秒计数
        val  = SysTick->VAL;     // 快照递减计数值
    } while (ms != g_millis);    // 防跨毫秒翻转竞态

    uint32_t load = SysTick->LOAD;  // CPUCLK_FREQ / 1000U
    uint32_t us_per_tick = CPUCLK_FREQ / 1000000U;
    return ms * 1000U + (load - val) / us_per_tick;
}
```

> 32MHz 下 `CPUCLK_FREQ/1000000 = 32`，每个 SysTick 计数值 = 1/32 μs。`timebase_micros()` 分辨率 ≈ 0.03μs，完全满足 T 法测量需求。

**编码器 GPIO 初始化**（encoder.cpp 中）：

```cpp
void Encoder::init()
{
    // 已由 syscfg 配置为 INPUT，这里只需使能中断
    NVIC_SetPriority(GROUP1_IRQn, 0);  // 最高 NVIC 优先级
    NVIC_EnableIRQ(GROUP1_IRQn);

    DL_GPIO_clearPins(GPIOA, DL_GPIO_PIN_15 | DL_GPIO_PIN_17);
    DL_GPIO_enableInterrupt(GPIOA, DL_GPIO_PIN_15 | DL_GPIO_PIN_17);
    DL_GPIO_setPinInterruptConfig(GPIOA, DL_GPIO_PIN_15 | DL_GPIO_PIN_17,
                                   DL_GPIO_INTERRUPT_EDGE_SENSITIVE_BOTH);
}
```

### 4.2 encoder.hpp 接口设计

```cpp
#pragma once
#include <cstdint>

// T 法测速：GPIO 中断捕获脉冲周期 → 计算 RPM
// 仅用 A 相单边沿，B 相用于方向判断（可选）
struct Encoder {
    void init();           // 初始化 GPIO 中断（PA15 或 PA17）
    void update();         // 每 10ms 调用，超时检测（无脉冲超过阈值→speed=0）
    float get_speed_rpm(); // 获取当前电机转速 (RPM)
    int32_t get_ticks();   // 获取累计脉冲数
    void reset();          // 清零计数

    // ISR 回调：由 GPIO 中断调用
    void on_pulse(uint32_t now_us);

    float    speed_rpm = 0.0f;
    int32_t  pulse_count = 0;
    uint32_t last_pulse_us = 0;   // 上次脉冲时间戳
    uint32_t pulse_period_us = 0; // 最近一次脉冲周期
};

extern Encoder g_enc_left;   // 左轮编码器 (PA15)
extern Encoder g_enc_right;  // 右轮编码器 (PA17)
```

### 4.3 速度 PID 参数

速度环使用**增量式 PID**（更平滑），参数独立于方向 PID：

| 参数 | 建议初值 | 说明 |
|------|----------|------|
| speed_kp | 0.5 | 速度比例系数 |
| speed_ki | 0.02 | 速度积分系数（补偿静差） |
| speed_kd | 0.0 | 速度微分系数（初值关闭） |
| speed_sum_limit | ±500 | 积分限幅 |
| speed_out_limit | ±1000 | 输出限幅（PWM 范围） |

> 调试顺序：先 KP（响应速度）→ KI（消除静差）→ KD（抑制过冲）

### 4.4 main.cpp 主循环修改点

```
每 10ms（现有控制周期）:
  1. g_line_sensor.update()              // 不变
  2. g_enc_left.update()                 // 新增：T 法超时检测 + RPM 保持
  3. g_enc_right.update()                // 新增
     （实际脉冲到达由 GPIO ISR 异步处理，update 只做超时归零）
  4. proto_poll_commands()               // 不变
  5. proto_send_telemetry()              // 不变（payload 扩展）
  6. turn_fsm_update()                   // 不变

  7. 控制决策:
     如果循迹/电机关 → 停车
     直线:
       error = 传感器误差
       correction = g_pid.calc(-error)     // 方向 PID 不变
       if (!g_dir_pid_on) correction = 0;  // ★ DEN=0 时 bypass 方向环
       left_target  = baseSpeed + correction
       right_target = baseSpeed - correction
       → 等比例饱和（同现有 steering.hpp）

       // ★ 新增：速度环
#if ENABLE_ENCODER
       if (g_speed_loop_on) {
           left_pwm  = g_spid_left.calc(left_target  - g_enc_left.speed_rpm)
           right_pwm = g_spid_right.calc(right_target - g_enc_right.speed_rpm)
       } else
#endif
       {
           left_pwm  = left_target;
           right_pwm = right_target;
       }

     转弯: → 固定目标速度（内轮/外轮），加速度闭环
     丢线: → 停车
```

### 4.5 降级兼容

通过编译宏 `ENABLE_ENCODER` 控制：

```cpp
#if ENABLE_ENCODER
    // 速度闭环路径
    left_pwm  = g_spid_left.calc(left_target  - g_enc_left.speed_rpm);
    right_pwm = g_spid_right.calc(right_target - g_enc_right.speed_rpm);
#else
    // 原有开环路径（不改动）
    left_pwm  = left_target;
    right_pwm = right_target;
#endif
```

- `ENABLE_ENCODER = 0`：完全保持现有行为，100% 向后兼容
- `ENABLE_ENCODER = 1`：启用速度闭环

## 五、小程序协议扩展

### 5.1 新增命令

| 命令 | 示例 | 说明 |
|------|------|------|
| `SKP=<value>` | `SKP=500` | 设置速度 KP（值/1000，实际 0.5） |
| `SKI=<value>` | `SKI=20` | 设置速度 KI（值/1000，实际 0.02） |
| `SKD=<value>` | `SKD=0` | 设置速度 KD（值/1000） |
| `SEN=<0/1>` | `SEN=1` | 速度环使能开关 |
| `DEN=<0/1>` | `DEN=0` | 方向环使能开关（关掉后 correction=0，两轮等速，电机不停） |
| `SPD?` | 查询 | 返回当前左右轮速 `Lxxx Rxxx` |
| `ENC?` | 查询 | 返回编码器原始脉冲数 |

> 命名规则：S 前缀 = **S**peed loop，D 前缀 = **D**irection loop。`SEN`/`DEN` 与 `LTO`（循迹使能，关=停车）不同——`DEN=0` 仅置零 correction，电机和速度环继续运行，方便单独调试速度环。

### 5.2 遥测帧扩展

在现有 47 字节 TlmPayload 后面追加 4 字节（不破坏现有结构，小程序解析时按 len 字段判断）：

```
原有 47 字节 + [left_rpm: int16_t][right_rpm: int16_t]
                                     ↓
                               新 payload 长度: 51 字节
```

新增字段：

| 字段 | 类型 | 偏移 | 说明 |
|------|------|------|------|
| left_rpm | int16_t | 47 | 左轮实际转速（RPM），比例因子 0.1 |
| right_rpm | int16_t | 49 | 右轮实际转速（RPM），比例因子 0.1 |

### 5.3 状态 flags 扩展

TlmPayload.flags 中新增 bit3：

| Bit | 含义 |
|-----|------|
| bit0 | 保留 |
| bit1 | 循迹关 |
| bit2 | 电机关 |
| **bit3** | **速度环关（新增）** |
| **bit4** | **方向环关（新增）** |

### 5.4 小程序适配要点

> **好消息**：小程序前端已内建编码器 + 速度环 UI 支持（`hasEncoder`、`encFL/FR/BL/BR`、`speedL/R`、`encoderPpr` 配置、速度环开关等）。MCU 端只需：

## 七、兼容性与安全性

### 7.1 不影响现有功能

| 功能 | 保障措施 |
|------|----------|
| 方向 PID | 不变，仍然输出 correction |
| 差速转向 | 不变，steering.hpp 饱和逻辑保留 |
| 直角弯 FSM | 不变，仍使用固定内/外轮目标速度 |
| 标定流程 | 不变，传感器标定零改动 |
| UART 协议 | 新增命令，不删不改旧命令 |
| 小程序界面 | 向后兼容，旧版 APP 忽略新增字段 |
| 编码器未接 | `ENABLE_ENCODER=0` 编译，完全回退到开环 |

### 7.2 安全边界

1. **速度反馈丢失检测**：连续 N 个周期 `delta_ticks == 0` 但 PWM 非零 → 编码器故障，自动切回开环或停车
2. **积分防饱和**：速度 PID 独立积分限幅，不污染方向 PID
3. **RST 命令**：同时重置方向 PID 和速度 PID，速度 PID 重置时调用 `set_sum_error(0)` 清零积分

### 7.3 调参顺序（重要）

```
Step 1: 编码器采集验证（SPD?/ENC? 命令查看原始值）
Step 2: KP 粗调（从 0.3 往上加，观察响应速度）
Step 3: KI 精调（从 0.01 往上加，消除稳态误差）
Step 4: KD 抑制过冲（从 0.01 开始）
Step 5: 与方向 PID 联调（先低 baseSpeed 再逐渐提速）
```

## 九、风险与注意事项

| 风险 | 缓解 |
|------|------|
| GPIO 与现有外设冲突 | 已确认 syscfg 中引脚无冲突 |
| **低 CPR（13 线）M 法失效** | 改用 T 法（周期测量），低速精度反而更高 |
| GPIO 中断频率过高 | 额定 400RPM 时 ~87Hz，远低于 MSPM0 承受上限 |
| 霍尔编码器电磁干扰 | 自带整形电路，且磁感应比光电抗干扰强 |
| 速度环震荡 | 从低 KP 起步，KI/KD 初值置零 |
| 两级 PID 耦合不稳定 | 方向 PID 输出变化率靠 KD 抑制，速度环带宽 > 2x 方向环 |
| 小程序兼容 | 新字段放在 payload 末尾，旧协议按 len 截断解析 |
| Flash/内存不足 | 编码器驱动和 PID 实例开销 < 500 字节，MSPM0 有 32KB SRAM |

## 十、总结

本方案在**不修改现有方向 PID、差速转向、直角弯 FSM、传感器标定、UART 基础协议**的前提下，通过以下方式增加速度闭环：

1. **硬件层**：新增双路编码器（GPIO + Timer 脉冲计数）
2. **控制层**：新增级联速度 PID（方向 PID 输出目标速度 → 速度 PID 输出 PWM）
3. **协议层**：新增 6 条小程序命令 + 遥测帧尾部扩展 4 字节
4. **兼容性**：`ENABLE_ENCODER` 编译开关，无编码器时完全回退到当前开环行为
